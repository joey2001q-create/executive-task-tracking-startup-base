# 日报模板

除非请求方要求其他语言，默认使用中文。

## 输出分层

最终输出必须分成两层：

- 聊天框：只给老板看清爽简报，不放工作明细。
- 飞书云文档：放高管工作详情，最终只把云文档链接返回到聊天框。

## 聊天框老板简报格式

不要在聊天框展示任务数量、消息数量、邮件数量、会议数量、原始证据 ID、长时间线或逐条工作明细。除非老板明确要求追溯证据，否则证据和明细只放进飞书云文档。

```
**🎯 关键事项**

- [核心工作2]

**✅ 关键决策**
- [决策内容；无则写「无」]

**‼️ 重大风险**
- [风险内容；无则写「无」]

- [需拍板事项；无则写「无」]

**🤔 本次管理风格** 
- [高管管理风格]

**📄 工作详情**  [https://...](https://...)

```

## 语气

- 聊天框简报：简洁、直接，对忙碌的 CEO 有用。
- 云文档详情：完整、可追溯，便于复盘和管理追踪。
- 两层输出都说明业务含义，而不只是罗列活动。
- 来源不完整时，区分“事实”和“推断”。

## 摘要长度

建议长度：

- 聊天框老板简报：1 屏到 2 屏内，按高管数量适度压缩。
- 今日最重要管理信号：3-5 条。
- 每位高管聊天框摘要：150-260 个中文字符，包含一句画像。
- 飞书云文档详情：不限于 1-3 页，但要按高管和主题组织，避免流水账。
- 数据缺口：两层都要覆盖；聊天框只放影响判断的重要缺口，云文档放完整缺口。

## 脱敏

替换敏感细节：

- 高管范围外的个人姓名：尽量改用角色或部门。
- 私人事项：`[已省略个人内容]`。
- 法律特权材料：`[已省略受限法律内容]`。
- 医疗、家庭、个人财务内容：从 CEO 简报中省略，除非已明确授权且业务上必须纳入。
