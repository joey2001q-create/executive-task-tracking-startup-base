# 高管横向对比

## 作用

当老板需要**同时看多个高管**的表现时，本 skill 做横向对比，帮老板快速识别：
- 谁在推进、谁在停滞
- 谁敢拍板、谁在回避决策
- 谁提前暴露风险、谁等爆雷才动
- 谁在带团队建机制、谁在单打独斗

## 输入

多位高管的数据（由 `feishu-data-collector-v3` 提供）：

```json
{
  "period": {"start": "...", "end": "..."},
  "executives": [
    {
      "name": "韩嘉辉",
      "open_id": "ou_xxx",
      "department": "AI研究院",
      "data": { /* data-collector 返回的完整数据 */ }
    },
    {
      "name": "李超",
      "open_id": "ou_xxx",
      "department": "商业化华南区",
      "data": { /* ... */ }
    }
  ]
}
```

## 对比维度（严格执行）

### 一、关键事项推进能力
**核心问题：他负责的事项有没有持续推进？**

对比点：
- 是否有明确目标？（日程/任务中有无清晰的目标描述）
- 是否有负责人？（任务分配是否明确）
- 是否有截止时间？（任务是否设定了 due_at）
- 是否有阶段结果？（妙记/群聊中是否有进展更新、完成信号）

**判断标准：**
- **强**：目标-负责人-时间-结果四要素齐全，且阶段结果可见
- **中**：有目标和时间，但结果模糊或滞后
- **弱**：目标不清、无人负责、时间模糊、长期无结果

### 二、决策能力
**核心问题：他有没有在关键节点拍板？**

对比点：
- 是否推动问题收敛？（单聊/群聊中是否出现"决定/按这个方案/定了"等信号）
- 还是让问题长期悬而未决？（同一议题反复讨论无结论）
- 决策是否有依据？（是否基于数据/客户反馈/团队讨论）
- 决策后是否有执行跟进？（决策后是否出现相应的任务/日程）

**判断标准：**
- **强**：关键节点主动拍板，有依据，有跟进
- **中**：有讨论有倾向，但未明确拍板
- **弱**：回避决策，议题反复悬置

### 三、风险识别和处理能力
**核心问题：风险是被他提前发现，还是等爆雷后才被动应对？**

对比点：
- 是否提前暴露风险？（群聊/单聊中是否主动提及"风险/可能/担心/注意"）
- 是否组织处理？（发现风险后是否拉会/分配任务/升级）
- 是否被动应对？（风险爆雷后才出现相关讨论/补救）
- 是否有风险闭环？（风险处理后是否有跟进确认）

**判断标准：**
- **强**：提前识别+主动组织处理+闭环确认
- **中**：发现风险但未主动处理，或处理方式被动
- **弱**：风险爆雷后才被动应对，或无风险意识

### 四、团队管理能力
**核心问题：他是否在带人、建机制、分责任？**

对比点：
- 是否在带人？（1:1、辅导、反馈、面试、新人培训等信号）
- 是否在建机制？（流程文档、规范、SOP、自动化工具等）
- 是否在分责任？（任务分配、RACI、授权等）
- 团队是否有成长信号？（新人转正、成员晋升、能力输出）

**判断标准：**
- **强**：带人+建机制+分责任，团队有成长信号
- **中**：有带人或分责任，但机制建设薄弱
- **弱**：单打独斗，团队无成长信号

## 输出标准

### L1 聊天框简报（飞书原生格式）

**老板允许在横向对比简报中使用 markdown 表格**，用于展示多高管对比矩阵。

```
**📊 高管横向对比 · [部门/公司] · [周期]**
**👤 对比对象**：高管A（部门）、高管B（部门）...

**🎯 一、关键事项推进能力**

| 高管   | 评级 | 依据 |
| ------ | ---- | ---- |
| 高管A  | 强   | 目标-负责人-时间-结果四要素齐全 |
| 高管B  | 中   | 有目标和时间，但结果模糊 |

**✅ 二、决策能力**

| 高管   | 评级 | 依据 |
| ------ | ---- | ---- |
| 高管A  | 强   | 关键节点主动拍板，有依据有跟进 |
| 高管B  | 弱   | 回避决策，议题反复悬置 |

**⚠️ 三、风险识别和处理能力**

| 高管   | 评级 | 依据 |
| ------ | ---- | ---- |
| 高管A  | 强   | 提前识别+主动组织处理+闭环确认 |
| 高管B  | 弱   | 风险爆雷后才被动应对 |

**👥 四、团队管理能力**

| 高管   | 评级 | 依据 |
| ------ | ---- | ---- |
| 高管A  | 强   | 带人+建机制+分责任，团队有成长 |
| 高管B  | 中   | 有带人，但机制建设薄弱 |

**🏆 综合判断**
- 本周表现最强：【姓名】+ 一句话原因
- 最需要关注：【姓名】+ 一句话原因

**📄 对比详情**  [https://...](https://...)
```

**写作要求：**
- 标题和维度名加粗 `**`
- **允许使用 markdown 表格**展示多高管对比矩阵（老板已批准）
- 表格列：高管 | 评级 | 依据
- 评级必须给出 "强/中/弱"，不能模糊
- 综合判断必须有"最强"和"最需关注"

### L2 云文档详情

包含：
- 每位高管的 4 维度详细分析（含证据 ID）
- 横向对比矩阵表
- 数据覆盖说明
- 老板建议关注事项

用 bot 身份创建：`lark-cli docs +create --api-version v2 --title ... --markdown @文件 --profile {{APP_ID}}`

## 使用方式

### 命令行

```bash
feishu-executive-comparison compare \
  --executives "韩嘉辉:ou_a033...,李超:ou_d8fb..." \
  --start "2026-06-09T00:00:00+08:00" \
  --end "2026-06-17T23:59:59+08:00" \
  --output /tmp/comparison_report.json
```

内部流程：
1. 对每个高管调用 `feishu-data-collector-v3` 获取数据
2. AI 按 4 维度对比分析
3. 生成 L1 简报 + L2 云文档

### 依赖

- `feishu-data-collector-v3`
- `lark-cli`
