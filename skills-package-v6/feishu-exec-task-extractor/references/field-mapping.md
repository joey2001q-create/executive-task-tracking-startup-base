## 批量写入性能

- **单条写入**：< 200ms
- **批量写入**：飞书 API 支持 `records` 数组一次最多 1000 条，建议每批 50 条
- **大批量同步**（如 100+ 新增任务）：分批写入，每批间隔 200ms，避免触发限流

```bash
# 批量新增（推荐）
lark-cli --profile <AppID> --as bot api POST \
  /open-apis/bitable/v1/apps/<APP>/tables/<TABLE>/records/batch_create \
  --data '{"records":[{"fields":{...}}, {"fields":{...}}]}'
```

> ⚠️ 实际上飞书 bitable 没有 batch_create 接口，需要用循环单条写入 + 适当 sleep
