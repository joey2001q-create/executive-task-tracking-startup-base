# AI Project Startup Base Checklist

这是一份给 AI Agent 的任务收尾检查清单。用于防止 Agent 在多轮对话后忘记更新项目记忆、约束、进展和交接文档。

本清单属于项目启动前的底座规范。具体项目如果约定只启用高管追踪和任务追踪两个 skill，应在收尾时额外确认 skill 边界是否仍然成立。

每次完成重要任务前，AI Agent 必须逐项检查。

## 1. 长期记忆

是否需要更新 `PROJECT_MEMORY.md`？

需要更新的情况：

- 产品长期方向变化
- 目标用户变化
- 用户长期偏好变化
- 项目计划启用的 skill 范围变化
- 跨项目/跨会话都应保留的协作原则变化

## 2. 当前上下文

是否需要更新 `PROJECT_CONTEXT.md`？

需要更新的情况：

- 当前项目状态变化
- 当前任务清单、跟进节奏或交接重点变化
- 品牌、仓库、发布、自动更新、部署策略变化
- 重要配置变化
- 新增或调整关键文件入口
- 后续 Agent 接手时必须知道的信息变化

## 3. 决策记录

是否需要更新 `DECISIONS.md`？

需要更新的情况：

- 做出关键产品或技术决策
- 明确放弃某个替代方案
- 形成长期约束
- 明确只保留或调整某些 skill 的范围
- 用户明确说明“以后都这样”
- 涉及架构、数据、安全、发布、品牌、UI 基调等影响较大的选择

## 4. 项目入口

是否需要更新 `README.md`？

需要更新的情况：

- 启动方式变化
- 安装依赖方式变化
- 构建/测试/发布方式变化
- 项目说明变化
- 新成员或新 Agent 的入口说明变化

## 5. 版本变化

是否需要更新 `CHANGELOG.md`？

需要更新的情况：

- 新增功能
- 修复问题
- 改变用户可见行为
- 改变发布配置
- 改变品牌/UI/主题
- 新增重要文档或协作规范

## 6. Skill 范围

项目计划启用的 skill 是否发生变化？

需要更新的情况：

- 用户要求新增或恢复其他 skill
- 项目原本约定只使用高管追踪和任务追踪，但边界发生变化
- 某个任务实际已经超出当前 skill 的职责范围

## 7. 最终回复格式

最终回复中必须包含文档更新说明：

```text
文档更新：已更新 PROJECT_CONTEXT.md、CHANGELOG.md；无需更新 PROJECT_MEMORY.md、DECISIONS.md、README.md，因为本次没有改变长期方向、关键决策或项目入口。
```

如果不确定是否需要更新，先问用户。
